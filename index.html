创建TCP连接的三次握手以及TCP协议分析

第一次握手 查看如下图所示的第一个TCP报文段内容

源端口号为4307，目的端口号为80，初始序号为0，数据段长度为0。

FLAG字段为0x002,SYN比特置为1，表示客户端想要与服务器建立一个TCP连接。

首部长度为28个字节，因为选项字段占用了8个字节，选项字段中客户端与服务器协调MSS为1460字节。

第二次握手查看如下图所示的TCP报文段内容

window size value字段表明服务器接收窗口大小为5850字节

服务器初始序号为0，ACK：1表明服务器期待接受的下一个客户端发送的TCP报文段序号为1.确认了第一次握手发送的报文。

第三次握手查看如下图所示的TCP报文段内容

最后一次握手这个报文段。序号为1，ACK为1表明客户端确认了服务器允许连接的报文段。因为连接已经建立了。所以SYN比特置为0.

至此。三次握手结束。成功建立TCP连接。

网络层 - 幕布






网络层
  • 概述
    • 2个主要功能：转发与路由选择routing
      • 转发表
        • 路由选择算法
        • 路由选择协议报文
      • 第三种功能：连接建立connection setup
    • 网络服务模型
      • 网络层可以提供的服务有
        • 确保交付
        • 具有时延上界的确保交付
        • 有序分组交付
        • 确保最小带宽
        • 确保最大时延抖动
        • 安全性服务
      • 因特网的网络层只提供尽力而为服务
      • ABR ATM网络服务
      • CBR ATM网络服务
  • 虚电报和数据报网络
    • 网络层：主机到主机的连接或无连接服务
    • 虚电报：连接服务。数据包：无连接服务
    • 虚电报网络 virtual-Circuit
      • 许多其他网络体系结构例如ATM是VC网络
      • VC的组成
        • 1.源和目的地的一系列路径
        • 2.VC号，沿着该路径的每段链路的一个号码
        • 3.沿着该路径的每个路由器的转发表
      • 每个链路不使用相同的VC号的原因见课本P208
      • ！VC网络中的路由器必须为进行中的连接维维持连接状态信息
        • 每当跨越一个路由器创建一个新连接，转发表增加一个表项，释放一个连接，删除表项
      • 3个阶段
        • 虚电路建立（掌握具体细节见课本P208）
        • 数据传送
        • 虚电路拆除
      • 网络层虚电路的建立与运输层的连接建立的不同
      • signaling message and signaling protocol
        • 端系统向网络发送指示虚电路启动与终止的报文以及路由器之间传递的用于建立虚电路的报文

为什么学生不喜欢上学

人天生就有好奇心，但我们不是天生的杰出思想者，除非认知环境复合一定的要求，否则我们会尽可能的避免思考

大脑尽可能的的躲避思考

  • 一些最重要的功能不需要思考
  • 人体倾向于使用记忆而不是思考指导行动,不断的重复同一项任务，会内化为习惯

好奇心与生俱来，但很脆弱

  • 脑力活动吸引我们因为给了我们体验成功的愉悦的机会，但不是所有的脑力活动都吸引我们
  • 是内容驱动我们的兴趣，问题的困难程度影响了我们对于不同脑力活动的看法
  • 思考是缓慢的，费力的，不确定的
