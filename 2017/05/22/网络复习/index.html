计算机网络复习四-网络层

Posted on 2017-05-22 |


网络层 - 幕布






网络层
  • 概述
    • 2个主要功能：转发与路由选择routing
      • 转发表
        • 路由选择算法
        • 路由选择协议报文
      • 第三种功能：连接建立connection setup
    • 网络服务模型
      • 网络层可以提供的服务有
        • 确保交付
        • 具有时延上界的确保交付
        • 有序分组交付
        • 确保最小带宽
        • 确保最大时延抖动
        • 安全性服务
      • 因特网的网络层只提供尽力而为服务
      • ABR ATM网络服务
      • CBR ATM网络服务
  • 虚电报和数据报网络
    • 网络层：主机到主机的连接或无连接服务
    • 虚电报：连接服务。数据包：无连接服务
    • 虚电报网络 virtual-Circuit
      • 许多其他网络体系结构例如ATM是VC网络
      • VC的组成
        • 1.源和目的地的一系列路径
        • 2.VC号，沿着该路径的每段链路的一个号码
        • 3.沿着该路径的每个路由器的转发表
      • 每个链路不使用相同的VC号的原因见课本P208
      • ！VC网络中的路由器必须为进行中的连接维维持连接状态信息
        • 每当跨越一个路由器创建一个新连接，转发表增加一个表项，释放一个连接，删除表项
      • 3个阶段
        • 虚电路建立（掌握具体细节见课本P208）
        • 数据传送
        • 虚电路拆除
      • 网络层虚电路的建立与运输层的连接建立的不同
      • signaling message and signaling protocol
        • 端系统向网络发送指示虚电路启动与终止的报文以及路由器之间传递的用于建立虚电路的报文
    • 数据报网络
      • 无需建立虚电路，路由器不维护连接状态信息。一有分组直接推进网络中，尽管转发表中维护了转发状态信息
      • 转发表
        • 最长前缀匹配规则
        • 转发表通过路由选择算法进行修改。大概1-5min更新一次
          • 这将导致分组在通过网络走不同的路径并可能无序到达
    • 因特网服务模型对服务保证要求最少（没有！)的优点
  • 路由器工作原理：聚焦于网络层的转发功能
    • 路由器的组成部分
      • 输入端口
        • 转发表的副本存放在每一个输入端口
        • ”匹配加动作“抽象
      • 交换结构
        • 经内存交换
          • 不能并行
        • 经总线交换
          • 不能并行
        • 经互联网络交换
          • N个输入端口，N个输出端口，2N条总线
          • 可以并行转发多个分组，如果输出端口不同的情况下
      • 输出端口
        • 输出端口取出存放在输出端口内存中的分组并发送到输出链路上
      • 排队情况（见课本P217）
        • 输入端口排队
          • 线路前部阻塞Head-Of-the-Line HOL阻塞
        • 输出端口排队
          • 从排队分组选择一个来发送
            • packet scheduler
              • 例如FCFS，WFQ（加权公平排队）
              • 在提供服务质量保证方面起着重要作用
          • 当没有内存来缓存一个分组，丢弃到达的分组还是删除排队的分组
            • 主动队列管理
            • 随机早起检测算法
      • 路由选择处理器
  • IP协议：因特网中的转发和遍址
    • 因特网的网络层的三个组件
      • IP协议
      • 路由选择
      • 报告数据报中的差错和对某些网络层信息请求进行相应的设施，ICMP协议
    • 数据报格式
      • 一般为20字节的首部（如果不包含选项）
      • 数据包长度16比特。理论最大65535字节，一般不超过1K5字节
      • IP数据报分片
        • 以太网frame承载不超过1500字节的数据
        • 最大传送单元MTU：一个链路层frame承载的最大数据量
        • 数据包的重新组装工作在端系统而不是路由器中
        • 标识号，标志（最后一个片为0，其他为10),偏移字段（！以8字节块为单位）
        • 当且仅当数据包完全重构为初始IP数据报的时候，才会传递到上层运输层。否则会被丢弃
        • 缺点
          • 使路由器，端系统更加复杂
          • 可以生成DoS攻击
    • IPV4编址
      • 概述
        • IP地址与接口关联，而不是主机或者路由器
        • IP地址4个字节32比特，最多有2^32个地址
        • 全球因特网的每一个接口，必须有一个全球唯一的IP地址，在NAT后面的接口除外
        • 子网（P226）
        • 子网掩码
      • 无类别域间路由选择Classless Interdomain Routing CIDR
        • 地址聚合/路由聚合/路由摘要
      • 分类编址
        • IP广播地址255.255.255.255
        • 每一个类别是2^n-2个。因为有2个地址用作特殊用途
      • 获取一块地址
        • ISP从ICANN组织获取IP块地址，组织的网络管理员向ISP请求IP地址块
      • 获取主机地址：动态主机配置协议
        • 一旦组织获得IP地址块，可以为本组织内的主机与路由器接口分配IP地址，路由器IP地址通常手工配置，但是主机地址一般用Dynamic Host Configuration protocol DHCP来完成
        • DHCP4个过程
          • DHCP服务器发现
            • UDP分组中向67端口给发广播地址送发现报文。本主机源地址0.0.0.0
          • DHCP服务器提供
            • DHCP服务器接收到DHCP发现报文，用DHCP提供报文做出相应。采用IP广播地址。子网中可能有几个DHCP服务器，客户自己选择一个
          • DHCP请求
            • 客户从一个或多个服务器中选择一个，然后发送DHCP请求报文
          • DHCP ACK
            • 服务器用DHCP ACK对DHCP请求报文进行响应
          • 一旦客户收到DHCP ACK 交互就完成了。当租用期到了之后。DHCP提供机制允许客户更新对IP地址的租用
      • 网络地址转换NAT
        • 专用地址的地域realm
        • NAT使能路由器隐藏了家庭网络的细节
        • 端口号为16比特。支持超过6K个并行使用路由器广域网一侧IP地址的连接
        • NAT的抗议者
        • NAT对P2P的妨碍
      • UPnP（略，待看）
      • 因特网
      • 因特网控制报文协议ICMP
        • 通常认为是IP的一部分。虽然体系结构位于IP之上（因为ICMP承载在IP分组之中）
        • ping工作原理
        • traceroute工作原理
      • 网络安全
        • IDS与IPS
        • 防火墙
      • IPv6
        • 40字节首部
        • 任播地址
        • 与IPv4的不同
          • 不允许在路由器进行分片与重新组装
          • 首部校验和去掉了
          • 选项去掉了。但是选项字段可以是下一个首部
          • ICMPv6
        • 从IPv4到IPv6
          • 双栈
          • 隧道
      • IP安全
        • IPsec
          • 面向连接
  • 路由选择算法
    • 默认路由器，第一跳路由器
      • 与主机相连接的路由器，又叫做源路由器
    • 分类
      • 第一种分类方式
        • global routing algorithm
          • Link State algorithm
        • decentralized routing algorithm
          • Distance-Vector algorithm
      • 第二种分类方式
        • 静态
        • 动态
      • 第三种分类方式
        • 负载敏感
        • 负载迟钝
    • 链路状态路由选择算法
      • 每个节点向网络中所有其他节点广播链路状态分组
      • 复杂度O(n^2)
      • 震荡
    • 距离向量路由选择算法
      • 链路费用改变引起的选择环路（P252）
        • 无穷计数问题
      • 毒性逆转技术
    • LS与DV的比较
      • 报文复杂性
      • 收敛速度
      • 健壮性
    • 层次路由选择
      • 原因；规模变大，组织希望自治
      • 自治系统AS
        • 处在相同管理下的路由器组成。运行相同的路由选择算法
          • 每个AS通常包含多个子网
        • 网关路由器：一个AS中负责向本AS之外的目的地转发分组的路由器
      • 自治系统间路由选择协议
      • ISP与AS的关系P257
        • 大部分ISP中的路由器以及互联他们的链路构成了单个AS
        • 许多ISP将网络划分成多个AS
    • 因特网中的路由选择
      • 因特网自治系统内部的路由选择：RIP
        • 被设置在下层ISP与企业网中
        • 内部网关协议 interior gateway protocol
      • 因特网自治系统内部的路由选择：OSPF
        • 被设置在上层的ISP中
        • 路由器向AS内所有其他路由器广播路由选择信息
        • 当链路状态变化时，路由器广播链路状态信息，即使没有变化。周期性地广播链路状态
        • 由IP承载，对OSPF其上层协议值为89
      • 自治系统间的路由选择：BGP 见P262
        • Broder Gateway Protocol
        • 聚合前缀，广播
        • 剩余部分略
