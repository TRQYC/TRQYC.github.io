1.HTTP，TCP，IP协议的分析

实验前的准备

打开浏览器，键入URL：http://gaia.cs.umass.edu/wireshark-labs/HTTP-wireshark-file4.html。

我们得到如下图所示的网页界面。正如网页上所说。这个网页包含2个图片

同时，我们的Wireshark抓包工具抓取了如下图所示的信息

创建TCP连接的三次握手以及TCP协议分析

第一次握手 查看如下图所示的第一个TCP报文段内容

源端口号为4307，目的端口号为80，初始序号为0，数据段长度为0。

FLAG字段为0x002,SYN比特置为1，表示客户端想要与服务器建立一个TCP连接。

首部长度为28个字节，因为选项字段占用了8个字节，选项字段中客户端与服务器协调MSS为1460字节。

第二次握手查看如下图所示的TCP报文段内容

window size value字段表明服务器接收窗口大小为5850字节

服务器初始序号为0，ACK：1表明服务器期待接受的下一个客户端发送的TCP报文段序号为1.确认了第一次握手发送的报文。

第三次握手查看如下图所示的TCP报文段内容

最后一次握手这个报文段。序号为1，ACK为1表明客户端确认了服务器允许连接的报文段。因为连接已经建立了。所以SYN比特置为0.

至此。三次握手结束。成功建立TCP连接。

HTTP协议分析

源IP为192.168.1.102，目的IP为128.119.245.12

分析HTTP报文内容（如上图所示）

GET方法字段指明了客户端要请求的资源URL，Host字段内容表明服务器主机是gaia.cs.umass.edu，User-Agent字段提供了客户主机的一些信息，浏览器默认接受的语言是en-us，HTTP版本是HTTP／1.1。保持TCP连接不断开

分析服务器返回给客户端的报文（如下图所示）。

服务器返回的状态码是200表明请求成功。查看数据部分内容。

客户端为这2个图片并行设置TCP连接。如下图所示。端口分别为4308与4309

IP协议分析

我们以服务器发送给客户端的第一次响应HTTP报文为例。分析其IP协议内容。报文内容如下

IP版本为4，头部长度20字节。总长度1043字节。标识符为29771，标志字段为0x744b表明此IP不分段。偏移值为0，存活跳数为55，上层协议报文为TCP。
